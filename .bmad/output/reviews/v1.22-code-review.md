- **mkdocs strict mode unverified locally.** `mkdocs` is not installed on
  the execution machine, so `mkdocs build --strict` was not run.
  CI will run it; the added markdown uses only constructs already present
  in other pages on the site (relative image paths, `{ loading=lazy }`
  attribute-list syntax, inline `<figcaption>` which requires the
  `md_in_html` extension — already present in `mkdocs.yml` line 90).
  **Risk: low.** Broken links / missing images would surface in CI.
- **Playwright scaffolding depends on a running demo backend.** The
  `npm run docs:screenshots` task requires a live `make demo` instance
  at `http://localhost:8080`. This is documented in the commit body and
  the script header comment. The script is not wired into CI — it's a
  developer-local refresh tool. **Acceptable.**
- **`internal/demo/mock_git.go` bootstrap Chart.yaml.** The added fixture
  keeps demo mode out of the FirstRunWizard by producing a bootstrapped
  repo state. This is a demo-only code path (guarded by the demo mode
  flag); no production risk. **Clean.**
