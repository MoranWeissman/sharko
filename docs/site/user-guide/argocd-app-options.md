# Editing ArgoCD Application Options

Starting in **v1.20.1**, the Addon Detail page has a dedicated **ArgoCD App Options** tab for editing the ArgoCD Application options that control HOW an addon is deployed (not what's inside its `values.yaml`).

## What's in ArgoCD App Options vs. Values

| Lives in ArgoCD App Options tab | Lives in Values tab |
|---------------------------------|---------------------|
| Sync wave (deploy order) | Helm `values.yaml` contents |
| Self-heal on/off | Per-cluster value overrides |
| Sync options (e.g. `ServerSideApply=true`) | — |
| Ignore differences | — |
| Additional sources | — |

A small header on each tab cross-links to the other — so if you open ArgoCD App Options looking for Helm values you'll see a *"Helm values? See the Values tab →"* hint, and vice-versa.

## Editing

1. Open the addon detail page (Addons → click any addon).
2. Click the **ArgoCD App Options** tab in the left rail.
3. Click **Edit** in the ArgoCD Application Options card header.
4. Change the fields you want:
   - **Sync Wave** — integer. Negative values deploy earlier (useful for CRDs); positive values deploy later. Zero is the default.
   - **Self-Heal** — toggle. When on, ArgoCD reverts manual cluster-side changes to match Git.
   - **Sync Options** — comma-separated ArgoCD sync options (`CreateNamespace=true, ServerSideApply=true`, etc.).
   - **Ignore Differences** — YAML list of fields ArgoCD should ignore during diff. Example:
     ```yaml
     - group: apps
       kind: Deployment
       jsonPointers:
         - /spec/replicas
     ```
   - **Additional Sources** — YAML list of extra chart or path sources for multi-source applications.
5. Click **Save (opens PR)**. Sharko opens a PR against `addons-catalog.yaml` with just the fields you changed (merge semantics — blank fields are left alone).

As with every Tier 2 mutation, if you haven't configured a personal GitHub PAT, the editor shows a yellow **attribution nudge** before you save, and the resulting commit is authored by the Sharko service account with you listed as a `Co-authored-by:` trailer. Set up a PAT in **Settings → My Account** to have your own name on these commits.

## Why separate from Values

ArgoCD Application options and Helm values live in different files in Git (`addons-catalog.yaml` vs. `addons-global-values/<addon>.yaml`) and affect different layers of the deploy pipeline:

- **ArgoCD App Options** change the ArgoCD Application generated by the ApplicationSet — sync wave, sync options, ignoreDifferences.
- **Values** changes the inputs to the Helm chart the ArgoCD Application renders.

Splitting them keeps each tab's diff focused and makes PR reviews easier — ArgoCD App Options PRs touch the catalog file; values PRs touch the values file.

## Read-only fields (for now)

A few addon fields aren't editable from the ArgoCD App Options tab yet — they live in the catalog file but are driven by other flows:

- **Version** — edited via the **Upgrade** tab (which runs an upgrade analysis first).
- **Dependencies (`dependsOn`)** — not yet surfaced in the UI. Edit the catalog file directly via the repo.
- **Namespace** — set at addon-add time.

If you need to change one of the read-only fields, edit `addons-catalog.yaml` in GitHub or your Git client. These are on the v1.21+ roadmap.
