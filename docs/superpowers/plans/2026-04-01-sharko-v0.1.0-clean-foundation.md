# Sharko v0.1.0 — Clean Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Strip dead code, rename module path, rebrand from "ArgoCD Addons Platform" to "Sharko". Same functionality minus stripped parts, new identity.

**Architecture:** No architectural changes. This is a rename/rebrand/cleanup pass. The existing server (`sharko serve`) works identically to the old `aap-server` binary. Cobra is introduced as the CLI framework to enable future subcommands.

**Tech Stack:** Go 1.25.8, React 18 + Vite, Helm 3, Docker multi-stage build, Cobra CLI

**Spec:** `docs/superpowers/specs/2026-04-01-sharko-implementation-design.md` — Sections 3 and 4 (v0.1.0)
